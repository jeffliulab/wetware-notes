# 笔记流水线 — 高质量笔记写作规范

> **本文件是本仓库所有新写 / 重写笔记的单一规范源。** 它与「心智 Mind · 类脑智能研究站」四大知识库（神经科学 / 认知科学 / 干细胞与细胞培养 / 湿件计算）共用同一套写作标准，改编自 [neuro-sci-notes](https://github.com/jeffliulab/neuro-sci-notes) 的 canonical spec。
>
> **新人入站读这一份就够了**：11 组件清单 + 写作风格 + 图示规范 + 自检清单。

---

## 1. 适用范围与粒度

**适用对象**：原子概念文章（独立 `.md` 文件）必须满足下面"必须"项；综述 / index 页满足其子集（可省 worked example / code）。

**粒度规则（Granularity Rule）**：一个概念满足下面任一条 → 独立成文，否则并进综述：

> 能教 ≥ 1 小时（含数学 + 例子）；在权威教材里占整个一节 / 一章；站内被 ≥ 3 篇文章引用。

---

## 2. 11 组件清单（文章骨架）

每篇原子概念文章按以下 11 组件组织，"必须"项全部到位才能 commit。

| # | 组件 | 必须? | 检查标准 |
|---|---|---|---|
| 1 | **Header**（标题 / 1-2 句定位 / 难度 / 前置知识） | ✓ | H1 中文标题 + 副标题英文(italic) + 1 句"这篇讲什么" + 前置 2-3 篇链接 |
| 2 | **直觉引入**（类比 / 物理直觉 / 生活场景） | ✓ | formal definition 之前 ≥ 1 段，以小白能懂的类比开头 |
| 3 | **形式化定义**（含中英术语标注） | ✓ | 关键术语首次用 `**中文 (English, ABBR)**`；定义用 KaTeX |
| 4 | **历史 / 动机** | 推荐 | 谁提出 / 为什么 / 解决什么；引文带年份 + 作者 |
| 5 | **数学推导**（KaTeX，步骤完整） | 视主题 | 数学密集主题必须；不省步，关键 step 单独成块 |
| 6 | **Illustration**（≥ 1 张 Tufte SVG） | ✓ | ≥ 1 张符合 §4 的 SVG |
| 7 | **伪代码 + Python 实现**（含输出） | 推荐 | 算法类必须；代码可运行，附 print 输出注释 |
| 8 | **Worked Example**（具体数值走一遍） | 推荐 | 算法 / 公式密集文章必须 |
| 9 | **Common Pitfalls / 易错点** | 推荐 | 概念易混的主题必须 |
| 10 | **Related Concepts**（前置/后续/横向/竞争） | ✓ | ≥ 2 条跨节链接 |
| 11 | **References** | ✓ | ≥ 3 条；权威教材页码 / 论文 / 课程；**不引 CSDN / 知乎** |

## 3. 写作风格

### 3.1 中英混排
- 中文为主（站内主语言）；关键术语首次出现 `**前向算法 (Forward Algorithm, FA)**`
- 英文术语遵循英文大小写习惯，不全小写
- 双语 i18n（suffix）：`.md` 中文 / `.en.md` 英文是镜像文件

### 3.2 数学（KaTeX）
- 行内 `$f(x)=ax+b$`；块级 `$$ E=mc^2 $$`
- 推导不省步，多行用 `\begin{aligned}` 对齐
- 定义 / 定理用 admonition：
  ```markdown
  !!! abstract "Bayes 定理"
      $$ P(A \mid B) = \frac{P(B \mid A) P(A)}{P(B)} $$
  ```

### 3.3 代码
- 语言优先级 Python > PyTorch > 其他；可运行、注明依赖、附输出注释；代码块标语言以启用高亮。

### 3.4 链接
- 跨节用相对路径 `[X](../OtherSection/X.md)`，不用绝对路径；跨语言无需手指 `.en.md`（i18n 插件自动选）。
- 外部链接加说明：`[arXiv:1706.03762 Attention Is All You Need (Vaswani et al., 2017)](https://arxiv.org/abs/1706.03762)`
- **跨板块链接**：四大知识库互为补充，鼓励在 Related 里链到其他板块（如 wetware 的「DishBrain」链到 neuro 的「突触可塑性」、cog-sci 的「预测处理」）。

## 4. 图示（Tufte 风格 SVG）

每篇原子概念文章 ≥ 1 张 SVG。嵌入方式：

```markdown
<div class="diagram">
<svg viewBox="0 0 720 280" xmlns="http://www.w3.org/2000/svg"> ... </svg>
</div>
<p class="figure-caption">Figure 1 — 一句话直觉总结。</p>
```

### Tufte 铁律
- **少即是多**：不要装饰边框 / 阴影 / 3D
- **直接标注**：标签直接放在边/节点上，不用 legend 间接指
- **节点大小有意义**：高亮节点更大 / 用强调色
- **暗色自适应**：颜色用 CSS 变量（如 `var(--dia-stroke)` / `var(--dia-accent)`），不硬编码，dark mode 自动响应
- **图说一句话**：`figure-caption` 给一句直觉总结，不复述正文

## 5. 质量自检清单（Commit 前过）

**内容**
- [ ] "必须"项全到位（Header / 直觉 / 定义 / Illustration / Related / References）
- [ ] 数学无"易得 / 显然"跳步；关键 step 单独成块
- [ ] 代码可运行、附输出
- [ ] 跨节链接 ≥ 2 条；References ≥ 3 条（含 ≥ 1 条权威教材）

**图示**
- [ ] ≥ 1 张 Tufte SVG，用 CSS 变量配色，caption 一句话

**双语**
- [ ] `.en.md` 同步存在；数学 / 代码 / 路径不翻译
- [ ] 自然学术英语，非逐字翻译；中英结构镜像（H1/H2/H3 一一对应）

**技术**
- [ ] `mkdocs build` 无 warning
- [ ] commit message 写清"新增 / 重写 / 仅路径迁移"
